Definitions
• Plaintext: message a party sends/receives
• Plaintext space: set of all messages
• Ciphertext: the encoding of the message that is exchanged
• Ciphertext space: set of all possible ciphertexts for the plaintext space
• Encryption: the encoding process from plaintext to ciphertext
• Decryption: the decoding process ciphertext to plaintext
• Adversary: a malicious party trying to break the encryption

• An encryption scheme is a triple of three efficient algorithms S = (Gen, Enc, Dec).
  • Gen(params) : On being given input some parameters params, the algorithm outputs a key κ that belongs to a set of all possible keys κ ∈ K.
  • Enc(κ, msg ): On being given input κ ∈ K and message msg ∈ M, the algorithm outputs an encoding of the message under the key, denoted Encκ(msg ). Encκ(.) is a function that is specified by the scheme. The algorithm outputs a ciphertext ct = Encκ(msg ).
  • Dec(κ′, ct): On being given input κ′ ∈ K and a ciphertext ct ∈ C, the algorithm produces the message encoded in the ciphertext, i.e., msg = Dec_{κ′} (ct). The function Dec_{κ′} (.) is specified by the scheme.

Kerchoff’s Principle
• The cipher method must not be required to be secret, and it must be able to fall into the hands of the adversary without inconvenience
• Security relies on a “secret” key that is not known to adversary
• Arguments in favor of this principle
• Easier to keep key secret than algorithm
• Easier to change key than to change algorithm
• Standardization
• Ease of deployment
• Public scrutiny

Types of Encryption Schemes:
• Symmetric Key Systems: The encryption key is the same as the decryption key κ = κ′. Also known as private key encryption schemes.
• Asymmetric Key Systems: The encryption key and the decryption key are different (although correlated), i.e., κ ̸= κ′. Also known aspublic key encryption schemes.

Requirements of encryption schemes
• Correctness: The message that has been encrypted using the encryption algorithm can be correctly decrypted using the decryption algorithm: Decκ′ (Encκ(msg)) = msg for all messages in M.
• Efficiency: The algorithms Gen, Enc and Dec should be efficiently computable.

Types of adversaries
• Eavesdropper: Adversary only has a ciphertext, and attempts to guess the plaintext
• Known-plaintext: Adversary is given some plaintext-ciphertext pairs, and attempts to guess the plaintext for some unknown ciphertext
• Chosen-plaintext: Adversary chooses some plaintexts, and is provided the ciphertexts for all these plaintexts, encrypted under the same key. Then, adversary is given an unknown ciphertext, and attempts to guess the plaintext
• Chosen-ciphertext: Chosen plaintext capabilities + adversary chooses some ciphertexts, and is provided the plaintexts after decryption. Then, adversary is given an unknown ciphertext, and attempts to guess the plaintext.

Perfectly secret encryption
• An encryption scheme over a message space M is perfectly secret if for every probability distribution over M, every message msg ∈ M, and every ciphertext ct ∈ C
Pr[ C = ct | M = msg ] = Pr[ C = ct]
(= Pr[ M = msg | C = ct ] = Pr[ M = msg ] This is easy to prove using Bayes’ theorem (in the book).)

• Let A be the event that a message msg has been encrypted by the
user, and let B be the event that a ciphertext ct has been generated by the
encryption algorithm. A and B are independent events.

• For any eavesdropper adversary that we can construct, even with
unlimited amount of computational resources, the algorithm remains
secure
• The scheme guarantees information-theoretic security i.e., regardless
of whatever new statistical/mathematical tools will be invented in the
future, the scheme remains secure

Perfectly secret encryption: One time pad (OTP)
• Invented in 1917.
• Message space is set of all binary strings of length ℓ: M = {0, 1}ℓ
• Key space is set of all binary strings of length ℓ: K = {0, 1}ℓ
• OTP(Gen, Enc, Dec) :
  • Key generation selects a random ℓ-bit binary string
  • Gen(1ℓ) : returns κ $← K
  • Encryption computes a bit-wise XOR of message and key
  • Enc(κ, msg ): Output ct = msg ⊕ κ
  • Decryption computes a bit-wise XOR of message and key
  • Dec(κ, ct): Output msg = ct ⊕ κ

Pr[ C = ct | M = msg ] = Pr[ κ = msg ⊕ ct] = 1 / 2ℓ

Pr[ M = msg | C = ct ] 
  = (Pr[ C = ct | M = msg ] · Pr[ M = msg ]) / Pr[ C = ct]
  = (2^{−ℓ} · Pr[ M = msg ]) / (\sum_{msg \in M} 2−ℓ · Pr[ M = msg ])
  = Pr[M = msg] / (\sum_{msg \in M} Pr[ M = msg ])
  = Pr[M = msg]
                       
Limitations of one time pad
• Key can be used only once
  • ct1 = msg1 ⊕ κ, ct2 = msg2 ⊕ κ =⇒ ct1 ⊕ ct2 = msg1 ⊕ msg2
• Key length is the same size as the message.

• Let S be a perfectly-secret encryption scheme over message space M and key space K. Then, it must be that
        |K| ≥ |M|   
• Towards a contradiction, assume |K| < |M|
• Let Mct = {msg |msg = Dec(κ, ct), κ ∈ K}
• Since Dec(κ, ct) is deterministic and only outputs one result, |Mct | ≤ |K| < |M|
• There is some msg′ ̸∈ Mct such that Pr[ M = msg ′ C = ct] = 0 ̸= Pr[ M = msg ]

Shannon’s Theorem
• Let S = (Gen, Enc, Dec) be an encryption scheme with message space M such that |M| = |K| = |C|. Then, S is perfectly secret if and only if
  1. Every key κ is chosen uniformly out of all keys in K with probability 1/|K| by the algorithm Gen
  2. For every msg and every ct, there is a single key κ such that Enc_κ(msg) = ct

Perfect indistinguishability
An encryption scheme over a message space M is perfectly secret if and only if for every probability distribution over M, every message msg0, msg1 ∈ M, and every ciphertext ct ∈ C
    Pr[ C = ct | M = msg0 ] = Pr[ C = ct | M = msg1 ]
       
Intuition:
Let us assume that there is an adversary algorithm Aeav that knows the entire message space M, knows the user’s message distribution and knows the entire ciphertext space C.
We will ask this algorithm to provide two inputs msg0, msg1 to provide two inputs to our encryption algorithm, and prove perfect-secrecy for these inputs. The adversary algorithm can implement any logic to choose messages

Adversarial Indistinguishability
• Define security with a ”thought” experiment (aka security game)
• Challenger (C): the good guy
• Adversary (A): the bad guy
• Let there be an encryption scheme S = (Gen, Enc, Dec)
• The challenger will implement all the algorithms in S
• The adversary is trying to ”break” the scheme implemented by the challenger
• The adversary is an algorithm that the challenger does not control

EXP^{EAV}_{A_{eav}}, _S(n)
• Superscript tell you what kind of ”attack” this experiment defines
• Subscript tells you what kind of adversary
• Subscript tell you what system the adversary is attacking
• Any parameter for the scheme S is an input to the experiment - in
• encryption schemes this is typically the key length

Eavesdropper experiment
EXP^{EAV}_{A_{eav}}, _S (n)
• Aeav selects two messages msg0, msg1 ∈ M and sends it to C
• C selects a key κ = Gen(n) and a bit b $← {0, 1}
• C sends ct = Enc(κ, msgb) to Aeav : send ct = Enc(κ, msg0) if b = 0, otherwise send ct = Enc(κ, msg1)
• Aeav guesses b by looking at ct and outputs b′ ← {0, 1}
• Output 1 if b′ = b, otherwise output 0

Eavesdropper experiment
• We can also represent the experiment as an algorithm running on the challenger (equivalent to previous slide)
• κ ← Gen(1^n)
• msg0, msg1 ← A_{eav} (1^n, .)
• b $← {0, 1}
• ct ← Enc(κ, msg_b)
• b′ = A_{eav} (1^n, ct)
• Output 1 iff. b′ = b, otherwise 0.

Limitations of perfect secrecy
• Perfect secrecy is impractical due to the lower bound on key size
• May be unneccessary since real adversaries may have limited resources
• Can we achieve a (slightly) weaker definition of security which will be enough 99.9999999....% of the time?

Definition
A scheme is (t, ϵ)-secure if every PPT. adversary running at most t steps succeeds in breaking the scheme with probability at most ϵ

• Consider an encryption scheme with a n-bit key selected randomly from 2n keys such that we define security by showing that after performing t steps an adversary succeeds in correctly guessing the key with probability t / 2n.
• Imagine n = 120, that is 2120 keys.
• An adversary will continue making guesses until it finds the correct
one.
• Making 260 guesses (where each guess is a clock cycle) will take about 35 years on the fastest supercomputer!

Negligible Functions
• A function f is negligible if for every polynomial p(.) there exists a N such that for every n > N it holds that
        f(n) < 1 / p(n)
• The function negl1(n) + negl2(n) is also negligible.
• The function p(n) × negl(n) is also negligible in n where p(n) is a polynomial in n
  • A polynomial-time adversary can call a polynomial number of sub-routines each with advantage that is negl(n) and still end up with negligible probability of breaking a scheme.
    
Pseudorandomness
• Assume there is a polynomial time algorithm D that participates in the following experiments
  • Exp_X
    1. z ← X // sample from uniform distribution X
    2. Send z to distinguisher D
    3. Repeat 1 - 2 polynomial number of times
    4. Output 1 if D guesses X
  • Exp_Y
    1. z ← Y // sample from non-uniform distribution Y
    2. Send z to distinguisher D
    3. Repeat 1 - 2 polynomial number of times
    4. Output 1 if D guesses Y

Computational Indistinguishability
• The distributions X and Y are computationally indistinguishable if for all PPT. distinguishers D
     |Pr[Exp_X = 1] − Pr[Exp_Y = 1]| ≤ negl(...)
• The probability that a polynomial-time adversary is able to distinguish a random distribution from a pseudorandom distribution that is computationally-indistinguishable is very small!

Pseudorandom Generator
• Let ℓ(.) be a polynomial and let G be a deterministic polynomial-time algorithm such that upon any input s ∈ {0, 1}^n, algorithm G outputs a string of length ℓ(n). G is a pseudorandom generator if the following two conditions hold.
  1. Expansion: For every n, it holds that ℓ(n) > n.
  2. Pseudorandomness: For all probabilistic polynomial time distinguisher algorithms D, there exists a negligible function |Pr[ D(r ) = 1] − Pr[ D(G (s)) = 1]| ≤ negl(n)
• A distinguisher tries to guess if a string is randomly sampled from the set of all strings {0, 1}ℓ(n), or it is the output of G. Distinguisher does not know s.

PRG indistinguishability game
EXP^{PRG}_{A_{prg}}, _G(n)
• C samples a bit b $← {0, 1}
• When b = 0:
  • s ← {0, 1}^n
  • y = G(s)
• When b = 1:
  • y ← {0, 1}^{ℓ(n)}
• Send y to Aprg
• A_{prg} outputs b′
• Output 1 iff. b′ = b.

EXP^{PRG}_{A_{prg}}, _G(n) = 1/2 + ADV^{PRG}_{A_{prg}}, _G(n)

PRG pseudorandomness
A PRG G is secure if for all probabilistic polynomial-time adversaries Aprg,
ADV^{PRG}_{A_{prg}}, _G(n) ≤ negl(n)
Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1] ≤ 1/2 + negl(n)

How far are we from random?
• How many unique strings of length n-bits are there? Let S_n be the set of all such strings, |Sn| = 2n
• How many unique strings of length ℓ(n) bits are there? Let Sℓ be the set of all such strings, |Sℓ| = 2^{ℓ(n)}
• How many unique outputs can G have? Let S_G be the set of all such strings, |S_G | = |S_n| = 2n
• Probability a random string r ∈ G(s)
• Pr[r ∈ G (s)] = |S_G| / |S_ℓ| = 2^{n−ℓ(n)}
• If ℓ(n) = 2n, then Pr[G (s) ∈ Sℓ] = 2−n

Pseudo One-time pad
• Let G be a pseudorandom generator with expansion factor ℓ. A private key encryption scheme S for messages of length ℓ(n) can be defined as follows.
  • Gen(1^n): choose a key κ ∈ {0, 1}^n uniformly at random and output it as the key
  • Enc(κ, msg): on input msg ∈ {0, 1}^{ℓ(n)}, output ct := G(κ) ⊕ msg
  • Dec(κ, ct): on input a key κ ∈ {0, 1}n and a ciphertext ct ∈ {0, 1}^{ℓ(n)} output the plaintext message msg := G(κ) ⊕ ct
  
A Reduction Proof for Pseudo OTP
• Start with the assumption that there is no PPT. adversary such that Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1] > 1/2 + negl(n).
• Assume that there is an adversary A_{eav} that wins against the EXP^{EAV}_{A_{eav}}, _S(n) with probability 1/2 + ϵ(n), i.e., Pr[EXP^{EAV}_{A_{eav}}, _S(n)] = 1/2 + ϵ(n)
• Construct an adversary A_{prg} that uses A_{eav} as a sub-routine to win against EXP^{PRG}_{A_{prg}}, _G(n),
  ⇒ A_{prg} wins EXP^{PRG}_{A_{prg}}, _G(n) every time A_{eav} wins against EXP^{EAV}_{A_{eav}}, _S(n). Otherwise, it loses.
• Bound ϵ(n), i.e., show that if ϵ(n) is non-negligible, then we have successfully constructed A_{prg} that breaks our starting assumption.

Setting up the proof
In both cases, A_{eav} ’s view is identically distributed to EXP^{EAV}_{A_{eav}}, _S(n) under different instantiations of S. Since A_{prg} wins every time A_{eav} wins
1. Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1 | b = 0] 
 = Pr[EXP^{EAV}_{A_{eav}}, _S(n) = 1] 
 = 1/2 + ϵ(n)
2. Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1 | b = 1] 
 = Pr[EXP^{EAV}_{A_{eav}}, _S(n) = 1] 
 = 1/2 (because S' is One-time Pad)
3. ϵ(n) = Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1 | b = 0] - 
          Pr[EXP^{PRG}_{A_{prg}}, _G(n) = 1 | b = 1] 
        = |Pr[A_{prg}(G(s)) = 1] - Pr[A_{prg}(r) = 1]|
4. If ϵ(n) > negl(n) then |Pr[A_{prg}(G(s)) = 1] - Pr[A_{prg}(r) = 1]| > negl(n), which implies A_{prg} is a PPT distinguisher with non-negligible advantage against G.
5. If ϵ(n) ≤ negl(n) is negligible for all PPT, then Pr[EXP^{EAV}_{A_{eav}}, _S(n) = 1] ≤ 1/2 + negl(n)

Abstract Stream Cipher

Let G be a pseudorandom generator with expansion factor ℓ. A private key encryption scheme S for messages of length ℓ(n) can be defined as follows:
• Gen(1n): choose a key κ ∈ {0, 1}^n uniformly at random and output it as the key

An abstract encrytion scheme from a random function.
• Gen(1n): choose a random function f : {0, 1}^n → {0, 1}^n 
• Enc(f , msg ): on input msg ∈ {0, 1}n, output ct := f (msg )
• Dec(f −1, ct): on input a ciphertext ct ∈ {0, 1}n output the plaintext message msg := f −1(ct)
Secrecy against eavesdroppers: This scheme is secure against eavesdroppers because a random function will produce a random ciphertext for the message.

Pseudorandom function
Keyed functions
• Consider functions of the form F : {0, 1}^n × {0, 1}^n → {0, 1}^n. These are called keyed functions. The first argument is a key.
• A keyed function F : {0, 1}^n × {0, 1}^n → {0, 1}^n induces a non-uniform distribution on the set of all functions {0, 1}^n → {0, 1}^n
• We will sample pseudorandom functions using this non-uniform distribution. We will use the key to select a particular function using the distribution.
• There are 2n such keyed-functions (i.e., selecting the key fixes the function) F_k : {0, 1}^n → {0, 1}^n

Oracle
• We use a concept called an ”oracle” to formalize the security definition.
• The oracle either implements Fk (.) or a random function f (.)
• The adversary interacts with the oracle and asks queries of the form F_k(x)/f(x)
• Based on the answers to the queries, the adversary tries to guess whether the oracle has implemented a PRF or a random function

Pseudorandom function
An efficient, length-preserving, keyed function F : {0, 1}^n × {0, 1}^n → {0, 1}^n is a pseudorandom function if for all probabilistic polynomial time distinguishers D, there is a negligible function negl such that
|Pr[D^{F_k(.)}(1^n) = 1] - Pr[D^{f(.)}(1^n) = 1] | ≤ negl(n)
Where the first probability is taken over the choice of k ∈ {0, 1}^n and the randomness of the distinguisher algorithm, while the second probability is taken over a uniform choice of a truly random function.
The superscript shows that the distinguisher has ”oracle” access, i.e., the distinguisher can query the function at any inputs of choice.

Pseudorandom permutation
A bijective pseudorandom function.
Consider the keyed function F_k : D → R
• Every element D is mapped to a unique element in R and no two elements in D can be mapped to the same element in R
• Every element in R has a pre-image in D
• Bijective functions are invertible: F_k(x) = y ; F^{−1}_{k}(y) = x

An efficient, length-preserving, keyed permutation F : {0, 1}^n × {0, 1}^n → {0, 1}^n is a pseudorandom permutation if for all probabilistic polynomial time distinguishers D, there is a negligible function negl such that
  |Pr[D^{F_k(.)}(1^n) = 1] - Pr[D^{f(.)}(1^n) = 1] | ≤ negl(n)
Where the first probability is taken over the choice of k ∈ {0, 1}^n and the randomness of the distinguisher algorithm, while the second probability is taken over a uniform choice of a truly random function

Strong pseudorandom permutation
In our definition of PRPs, we have provided the distinguisher access only to the forward query oracle that computes Fκ(x). We can consider stronger definitions where the distinguisher is also provided access to a backward oracle which answer queries for the inverse F^{−1}_{κ}(y).
An efficient, length-preserving, keyed permutation
F : {0, 1}^n × {0, 1}^n → {0, 1}^n is a strong pseudorandom permutation if for all probabilistic polynomial time distinguishers D, there is a negligible function negl such that
  |Pr[D^{F_k(.), F^{-1}_k(.)}(1^n) = 1] - Pr[D^{F(.), F^{-1}(.)}(1^n) = 1] | ≤ negl(n)


Block ciphers

Block ciphers are encryption schemes that are build from strong pseudorandom permutations.
Let F : {0, 1}^n × {0, 1}^n → {0, 1}^n be a length-preserving keyed strong pseudorandom permutation. S = (Gen, Enc, Dec) is an encryption scheme.
• Gen(1^n): On input a security parameter, select a key κ uniformly for the pseudorandom permutation F .
• Enc(κ, msg): On input a message msg and a key κ, output ct = Fκ(msg )
• Dec(κ, ct): On input a ciphertext ct and a key κ, output msg = F^{−1}_κ(msg)

• For a truly random function f , let x and x′ be two arbitrary inputs that differ by one bit. Then f(x) and f(x′) are uncorrelated (except they are not the same).
• For a PRP, let F_κ(x) and F_κ(x′) be the outputs on x, x′. Then, we want
  • Pr[F_κ(x)[i] = Fκ(x′)[i]] ≃ 1/2
• Avalanche effect: A one bit change in the input should affect all the bits in the output

The confusion diffusion paradigm
• Compute F : {0, 1}^n × {0, 1}^n → {0, 1}^n using a set of smaller functions f_1 : {0, 1}^t → {0, 1}^t , . . . , f_j : {0, 1}^t → {0, 1}^t where t << n. The key is used to select the j smaller functions.
• Partition the input string x ∈ {0, 1}n into j t-bit strings 
  • x = x1||x2|| . . . ||xj
• The confusion step:
  • F_κ(x) = f1(x1)||f2(x2)|| . . . ||fj (xj)
• The diffusion step: Use a fixed mixing permutation to permute the bits in F_κ(x). This ”diffuses” the changes to the entire output.
• Multiple rounds: Each confusion-diffusion step is a round. Repeat sequentially for multiple rounds with different keys

Substitution permutation networks
An implementation of confusion diffusion paradigm. In each round, given x ∈ {0, 1}^ℓ do the following.
• Key mixing: Set x = x ⊕ k where k is the round key
• Substitution: Let x′ = x1|| . . . ||xj . Set x = S1(x1)|| . . . ||Sj (xj ). S1, . . . , Sj are lookup tables that are called S-boxes and permute the inputs.
• Permutation: Permute x′ using a fixed mixing permutation (not dependent on the key). Output x′′ for the round.
The mixing permutation and design of S boxes is known publicly.

Avalanche effect
• The avalanche effect from the design of the S-boxes and the mixing permutation
• Changing a bit in the input produces at least a 2-bit change in the output for each S-box.
• The mixing permutation ensures that the output of each S box is dispersed in such way that it impacts at least two S boxes in the next round.


IND-CPA

EXP^{IND−CPA}_{A_{cpa}}, _S(n)
1. C selects a key κ = Gen(n)
2. A_{cpa} is given input 1^n and oracle access to Enc_κ(.)
3. A_{cpa} selects two messages msg_0, msg_1 ∈ M and sends it to C
4. C selects a bit b $← {0, 1}
5. C sends ct = Enc(κ, msg_b) to A_{cpa}: send ct = Enc(κ, msg_0) if b = 0, otherwise send ct = Enc(κ, msg_1)
6. A_{cpa} guesses b by looking at ct and outputs b′ ← {0, 1}
7. Output 1 if b′ = b, otherwise output 0.
The adversary makes polynomial number of queries to Enc_κ(.) which may include queries for msg_0, msg_1 to Enc_κ(.).

What do we need to build CPA secure encryption schemes?
• We want our encryption algorithm to produce random mappings from message to ciphertext. A (pseudo)random function will do this for us.
• We want to include additional random bits per message so as to produce different ciphertexts for the same message
But doesn’t that mean we need a one-time pad?
• .... No, because we will introduce random bits in a way that can be made public to the adversary
• The random bits need not be as long as the message

CPA secure encryption scheme from PRF
Given a pseudorandom function F do the following
• Gen(1^n) choose a uniform string κ ∈ {0, 1}^n and output it.
• Enc (κ, msg): On input κ ∈ {0, 1}^n and a message msg ∈ {0, 1}^n, select a random string r ∈ {0, 1}^n output the ciphertext ct :=<r , Fκ(r ) ⊕ msg>.
• Dec(κ, ct): on input ciphertext c =< r , s >, output msg := F_κ(r) ⊕ s
• The random string r is not kept hidden from the adversary
• The adversary does not have κ and so cannot compute F_κ(r)

Encrypting related data
How do we encrypt multiple (potentially related) messages with block ciphers?
• We can use a randomly sampled initialization vector (IV) to generate initial state
• Link ciphertexts with each for related data 
• Different modes of operation depending on how we link ciphertexts

But what about integrity?
• How does Bob know that a ciphertext he has received is the one that Alice sent, and an adversary has not tampered with the message in transit?

What can an adversary do realistically?
• A ciphertext is a random string, altering some of the bits in the ciphertext should not result in something meaningful (or does it?)

Integrity is not authentication
• We are not concerned (for now) with proving that Alice is the one sending a message to Bob (and not someone else). That is what we will call (imprecisely) as authentication.
• We are interested in proving that the message received by Bob is the one sent by Alice. If Eve is spoofing Alice’s identity in some way, we do not care!

Chosen plaintext attack
Intuition: We will model the fact that an adversary can have an arbitrary number of plaintext-ciphertext pairs of their choice
How will the adversary get plaintext-ciphertext pairs?
• We will give the adversary ”oracle” access to the encryption algorithm, denote Enc_k (.). The adversary can provide a plaintext msg as input to the oracle and get back the ciphertext Enc_k (msg) = Enc(κ, msg).
• This is more powerful than only revealing the encryption algorithm to the adversary because the oracle also has the key.
• The adversary can query the oracle an arbitrary number of times, but polynomially many times for a PPT adversary.

Indistinguishability under chosen plaintext attack
EXP^{IND−CPA}_{A_{cpa}}, _S(n)
1. C selects a key κ = Gen(n)
2. A_{cpa} is given input 1^n and oracle access to Enc_κ(.)
3. A_{cpa} selects two messages msg_0, msg_1 ∈ M and sends them to C.
4. C selects a bit b $← {0, 1}.
5. C sends ct = Enc(κ, msg_b) to A_{cpa}: send ct = Enc(κ, msg_0) if b = 0, otherwise send ct = Enc(κ, msg_1)
6. A_{cpa} guesses b by looking at ct and outputs b′ ← {0, 1}
7. Output 1 if b′ = b, otherwise output 0
• The adversary can keep making queries to Encκ(.) locally which is not explicitly specified in the game
• The adversary can also make queries for msg_0, msg_1 to Encκ(.).
• CPA security is stronger than multi-message security

Computational security against CPA adversary
An encryption scheme S = (Gen, Enc, Dec) is secure under chosen-plaintext attacks (or is CPA secure) if for every probabilistic polynomial time adversary A
Pr[EXP^{IND−CPA}_{A_{cpa}}, _S(n) = 1] = 1/2 + ADV^{CPA}_{A_{cpa}}, _S(n) ≤ 1/2 + negl(n)
CPA security is the minimum level of security we require in an encryption scheme today

CPA secure encryption S = (Gen, Enc, Dec)
Given a pseudorandom function F do the following
• Gen(1^n) choose a uniform string κ ∈ {0, 1}^n and output it
• Enc (κ, msg): On input κ ∈ {0, 1}^n and a message msg ∈ {0, 1}^n, select a random string r ∈ {0, 1}^n and output the ciphertext ct :=<r , F_κ(r) ⊕ msg>
• Dec(κ, ct): on input ciphertext c =<r, s>, output msg := F_κ(r) ⊕ s
• The random string r is not kept hidden from the adversary
• The adversary does not have κ and so cannot compute Fκ(r)

Message Authentication Code
A message authentication code consists of three polynomial time algorithms (Gen, TagGen, Vrfy):
• Gen(1^n): takes as input a security parameter and outputs a key of length n, κ = {0, 1}^n
• TagGen(κ, msg) : takes as inputs a key κ and a message msg and produces a tag t ← MAC_κ(msg)
• Vrfy(κ, msg, t): takes as input a message msg, a key κ and a tag t and verifies that t is a valid tag for message msg, that is b = Vrfy_κ(msg , t) and b = 1 for a valid tag, and 0 otherwise
Typically, Vrfy_κ(msg , t) checks that t = MAC_κ(msg) but the definition also allows for algorithms where this may not be the case

MAC security experiment (Unforgeability)
EXP^{MAC −FORGE}_{A_{mac}, MAC}(n)
• The challenger generates a key by running Gen(1^n)
• The adversary A_{mac} is given 1^n and access to a MAC oracle MAC_κ(.). Let Q denote the set of all queries by the adversary to the oracle
• The adversary outputs a pair (m, t).
• The adversary wins (and the experiment outputs 1) if:
  1. Vrfy_κ(m, t) = 1
  2. A_{mac} has not queried MAC_κ(.) with m as input, i.e., m !∈ Q

Security of a MAC
A message authentication code MAC = (Gen, TagGen, Vrfy) is secure if for all probabilistic polynomial time adversaries Amac , there is a negligible function negl such that 
Pr[EXP^{MAC−FORGE}_{A_{mac}, MAC}(n) = 1] ≤ negl(n)

EXP^{MAC−FORGE}_{A_{mac}, MAC}(n) - strong 
• The challenger generates a key by running Gen(1^n)
• The adversary A_{mac} is given 1^n and access to a MAC oracle MAC_κ(.). Let Q denote the set of all queries by the adversary to the oracle along with the tags returned by the oracle.
• The adversary outputs a pair (m, t).
• The adversary wins (and the experiment outputs 1) if:
  1. Vrfy_κ(m, t) = 1
  2. A_{mac} has not produced a tag for a new message, and it has not produced a new tag for an old message, i.e., (m, t′) /∈ Q for some valid t′ ̸ = t
  
Difference universal function
A keyed function h_κ is ϵ(n)-difference universal if for all n, any distinct m, m′, and for some ∆ (a function of n) it holds that
Pr[h_κ(m) − h_κ(m′) = ∆] ≤ ϵ(n)
where the probability is over the choice of κ

MAC from difference universal functions
Let h be a difference-universal function, and let F be a PRF. 
• Gen(1n): on input a security parameter 1^n, choose keys κ_h for h and κ_F be the key for F
• TagGen(κh, κF , msg ) : on input a message msg, and keys for the universal difference function and the PRF, select a random string r ∈ {0, 1}^n and output the tag
    t = <r, h_κ_h(msg) + F_κ_F(r)>
• Vrfy(msg , κh, κF , t): On input a message msg , a tag t =<r, s> and keys for the universal difference function and the PRF, verify that
    h_κ_h(msg) + F_κ_F(r) = s
• Output 1 if the verification is successful, otherwise output 0

Unforgeable encryption
EXP^{ENC−FORGE}_{A_{UE}}, S(n)
• The experiment generates a key by running Gen(1^n)
• The adversary A_{UE} is given 1^n and access to an encryption oracle Enc_κ(.). Let Q denote the set of all queries by the adversary to the encryption oracle.
• The adversary outputs a ciphertext c.
• Let m = Dec_κ(c). The adversary wins (experiment outputs 1) if:
  1. m ̸= ⊥ i.e., m is a valid message
  2. A has not queried m to the encryption oracle i.e., m ̸∈ Q

Unforgeability of Encryption
An encryption scheme S = (Gen, Enc, Dec) is unforgeable if for all PPT. adversaries A, there is a negligible function negl such that
Pr[EXP^{ENC−FORGE}_{A_{UE}}, S(n) = 1] ≤ negl(n)

Authenticated encryption
A security property that simultaneously satisfies secrecy and integrity
An encryption scheme is an authenticated encryption if
  1 The encryption scheme is CCA secure
  2 The encryption scheme has unforgeable encryptions
  
Point 1 does not imply point 2.
A CCA secure encryption scheme may allow an adversary to forge new ciphertexts on behalf of the sender, but these ciphertexts do not provide any information about other message-ciphertext pairs.
Point 2 does not imply point 1.
Unforgeable encryption experiment does not provide the adversary access to a decryption oracle

Two different classes of Oracles
”Real World” Oracles:
• Enc_κ(·) returns ciphertexts corresponding to    messages encrypted under the key κ
• Dec_κ(·) returns messages    corresponding to ciphertexts decrypted under the key κ
”Ideal World” Oracles:
Enc^0_κ(·) returns ciphertexts corresponding to encryptions of 0.
Enc^0_κ(·) = Enc_κ(0^n)
Dec^⊥_κ(·) returns decryption failed regardless of input Dec⊥

If you encryption scheme allows the adversary to have access to the ”Ideal World” oracle, the adversary learns nothing!
• The ideal encryption oracle encrypts only 0s regardless of what the adversary queries
• The ideal decryption oracle always fails
An encryption scheme is an authenticated encryption if
1. The encryption scheme is CCA secure =⇒ the encryption oracle allows the adversary to win with only negligible probability =⇒ replacing the real encryption oracle with the ideal encryption oracle should be indistinguishable to the adversary.
2. The encryption scheme has unforgeable encryptions =⇒ the decryption oracle will accept a ciphertext and produce a valid message with only negligible probability =⇒ replacing the real decryption oracle with the ideal decryption oracle should be indistinguishable to the adversary

Authenticated Encryption

EXP^{AE}_{A_{AE}}, _S(n)
1. A key is generated by running Gen(1^n)
2. A uniform bit b ← {0, 1} is selected
3. The adversary A_{AE} is given input 1n and access to two oracle:
  • If b = 0, then adversary is given access to the real world oracles Encκ(·) and Decκ(·)
  • If b = 1, then adversary is given access to the ideal world oracles Enc0κ(·) and Dec⊥κ(·)
 • A_{AE} cannot query the decryption oracle with a ciphertext generated from the encryption oracle
4. AAE outputs a bit b′ and wins if b′ = b

AE definition
Computational security for Authenticated Encryption
An encryption scheme S = (Gen, Enc, Dec) is an authenticated encryption scheme if for every probabilistic polynomial time adversary A_{AE}
Pr[EXP^{AE}_{A_{AE}}, _S(n) = 1] ≤ 1/2 + negl(n)
This definition says that a real world authenticated encryption schemes behaves very similarly to an ideal encryption scheme (but not perfect!) which does not provide any advantage to the adversary through encryption and decryption oracle accesses

Key selection in authenticated encryption
Always use different keys for the CPA-secure encryption scheme, and the MAC scheme
Consider a scheme where π_E has the following encryption function Enc(κ, msg) = Fκ(msg ||r), where r is a random string, and F is a pseudorandom-permutation. If |r| = |msg|, then π_E is CPA-secure.
Consider a MAC scheme where MAC_κ(msg) = F^{−1}_κ(msg) – this is a secure MAC because if F is a PRP, then so is F^{−1}.
Now construct an authenticated encryption S = (πE , πM ). What does the encryption produce if the same key is selected for πE and πM?
c1 := Enc(κ, msg ) = Fκ(msg||r)
c2 := MACκ(c1) = F−1κ (Fκ(msg||r)) = msg ||r

CBC mode with padding
Encryption algorithm
• Let block cipher length be ℓ and message size be |m| for message m
• Divide the message into t = ⌊m / ℓ⌋ blocks m1, . . . , mt
• Let m_{t+1} be the last block where ℓ − |mt+1| = b bytes
• Append to mt+1 0xb 0xb 0xb 0xb...b times.
• Encrypt m1, . . . , mt+1 with a block cipher in CBC mode
Decryption algorithm
• The receiver decrypts each block, then checks the last byte.
• If the last byte is b, then it checks if b is repeated b times. If not, output a padding error
• If there is no padding error, remove the padding and output the message

Chosen ciphertext security (CCA)
How do we model padding-oracle like attacks and defend against them?

Eavesdropper: Only looks at ciphertexts in transit
Chosen plaintext adversary: Has access to an encryption oracle which allows queries and allows the adversary to get ciphertexts for messages of its choice encrypted under the key selected by the communicating parties

Chosen ciphertext adversary:
  1. Has access to an encryption oracle which allows queries and allows the adversary to get ciphertexts for messages of its choice encrypted under the key selected by the communicating parties
  2. Has access to a decryption oracle which allows it to decrypt any string (which may be a ciphertext) of its own choice under the key selected by the communicating parties
  
The CCA experiment
EXP^{IND−CCA}_{A_{cca}}, _S(n)
1. κ ← S.Gen(1n)
2. m0, m1 ← A^{Encκ(·),Decκ(·)}_{cca}(1n)
3. b ← {0,1}
4. ct = Enc_κ(m_b)
5. b′ ← A^{Encκ(·),Decκ(·)}_{cca}(ct) # A_{cca} cannot query Decκ(·) on ct
6 Output 1 if b′ = b

Computational security against CCA adversary
An encryption scheme S = (Gen, Enc, Dec) is secure under chosen-ciphertext attacks (or is CCA secure) if for every probabilistic polynomial time adversary A

Pr[EXP^{IND−CCA}_{A_{cca}}, _S(n) = 1] ≤ 1/2 + negl(n)  

CCA1 Security: temporary access to decryption oracle The adversary is provided access to the decryption oracle Decκ(·) until it receives the challenge ciphertext in step 5.
CCA2 Security: permanent access to decryption oracle The adversary is provided access to the decryption oracle Decκ(·)
indefinitely.

Let F : {0, 1}^n → {0, 1}^n be a pseudorandom function.
• Gen(1^n) choose a uniform string κ ∈ {0, 1}^n and output it
• Enc (κ, msg): On input κ ∈ {0, 1}^n and a message msg ∈ {0, 1}^n, select a random string r ∈ {0, 1}^n, and output the ciphertext ct :=<r, F_κ(r) ⊕ msg>.
• Dec(κ, ct): on input ciphertext c =<r, s>, output msg := Fκ(r) ⊕ s

Is this scheme CCA secure?
A_{cca}:
• Select two messages m_0 = 0^n (string of all 0s) and m_1 = 1^n (string of all 1s)

Hash functions
A cryptographic hash function h is a deterministic function with the following two properties.
• Compression: The hash function maps arbitrary-length strings to strings of a fixed length ℓ
• Collision resistance: It is hard for a polynomial time adversary to produce a pair of inputs x ̸= x′ such that h(x) = h(x′)

Hash-and-MAC
Use hash functions to produce MACs for arbitrary length messages
• Take any arbitrary length message m ∈ {0, 1}∗ and compute h(m) ∈ {0, 1}^ℓ
• Compute a MAC on h(m), i.e., t = MACκ(h(m))
What does an adversary need to do to break security of the MAC scheme?
Produce a tag t′ from some m′ (unseen before) without knowing κ
• The adversary can try to find m′ such that h(m′) = h(m). In this case t = MAC_κ(h(m)) = MAC_κ(h(m′)). But this requires the adversary to find a collision in h.
• If the adversary has not found a collision, then it has produced a tag on h(m′) which it has not seen before

Is it possible to build a MAC scheme with just a hash function?
General Idea:
• Compute an ”inner” hash of a message concatenated with some random string κin to produce a ”hashed message”
• Compute an ”outer” hash of the ”inner” hash concatened with some random string κout to produce a tag

What are cryptographic hash functions?
• A lot of cryptographic tools have fixed input types, e.g., keys are fixed-length strings, PRFs have specified input domains, etc. Application data may not match these input types, lengths, or other parameters (if any)
• How do we turn application data in arbitrary domains to inputs that can be used in cryptographic protocols?
Cryptographic hash functions (informally): a mechanism to compute ”digests” (or compressed representations) of inputs suitable to cryptographic applications without violating security guarantees

Cryptographic hashing
• Collision resistance (in a non cryptographic sense): Two different items produce the same digest (or hash code) with very small probability!
• Collision resistance (in a cryptographic sense): Given an item x and h(x), it is hard for a polynomial time adversary to find a collision, i.e., some x′̸ = x where h(x) = h(x′).
• There will be collisions because you are mapping n things to ℓ strings where ℓ < n, the challenge is to find such a collision in polynomial time when n and ℓ are very large!

Cryptographic hash function
• A cryptographic hash function is a keyed function but the key is available to the adversary and in general is public information. The key is used to select one particular function, out of all such possible functions.

Definition
• A cryptographic hash scheme is a pair of probabilistic polynomial time algorithms (Gen, H) satisfying the following
  • Gen(·) takes as input a security parameter and outputs a key s. This selects the function h^s(·) : {0, 1}^n → {0, 1}^ℓ out of all possible functions where ℓ < n
  • H(·) is a deterministic algorithm that takes as input a key s and a string x ∈ {0, 1}^n and outputs a string h^s(x) ∈ {0, 1}^ℓ  
  
Collision resistance
EXP^{CR}_{A_{CR}}, _H(n)
1. A key is generated by running Gen(1n)
2. The adversary A_{CR} is given the key for the hash function s, and outputs x, x′ ∈ {0, 1}^n
3 The experiment outputs 1 if and only if x ̸= x′; h^s(x) = h^s(x′). In this case, the adversary has found a collision in the hash function
Collision resistance
A hash function h^s(·) is collision resistant if and only if for all probabilistic polynomial time adversaries A, there is a negligible function negl such that

Pr[EXP^{CR}_{A_{CR}}, _H(n) = 1] ≤ negl(n)

If we uniformly select q ∈ Θ(2^{ℓ/2}) items to query the hash function h^s: {0, 1}^n → {0, 1}^ℓ, then the probability that we will get a collision is roughly 1/2.

If the adversary computes the hash function on q distinct random inputs then, the probability it finds a collision is roughly q^2 / 2^ℓ

Collision resistance implies that it hard for an adversary to find any pair of collisions for a hash function h.
