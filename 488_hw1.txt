Taking the logical OR of all bits cannot function like XOR. Consider

Enc_{OR}(\kappa = 01, msg = 00) = 01
Enc_{OR}(\kappa = 01, msg = 01) = 01
Enc_{OR}(\kappa = 01, msg = 10) = 11
Enc_{OR}(\kappa = 01, msg = 11) = 11

If our decoder function knows that \kappa = 01 and $ct$ = 11, there will be no way for the decryption algorithm to discern whether $msg = 10$ or $msg = 11$. In general, there is no way to decrypt bits where the key was 1. Similarly, a one-time pad with AND looks like this.

Enc_{AND}(\kappa = 01, msg = 00) = 00
Enc_{AND}(\kappa = 01, msg = 01) = 01
Enc_{AND}(\kappa = 01, msg = 10) = 00
Enc_{AND}(\kappa = 01, msg = 11) = 01

There is no way to decrypt bits where \kappa_n = 0, and thus this cannot function as an encryption scheme.

If the output string has an even amount of bits, then the first was chosen, and there will be either $2n$ or $2n+2$ bits activated depending on whether the chosen bit was active in the key. If there are an odd amount of bits activated, then the second was chosen.

An encryption scheme over a message space M is perfectly secret if for
every probability distribution over M, every message msg ∈ M, and every
ciphertext ct ∈ C, Pr[ C = ct | M = msg ] = Pr[ C = ct]. This is equivalent to
Pr[ M = msg | C = ct] = Pr[ M = msg ], by Bayes theorem.

Dec(\kappa = 0, $ct$=2) = 2

Dec(\kappa = 2, $ct$=2) = 4
Dec(\kappa = 3, $ct$=2) = 5
Dec(\kappa = 4, $ct$=2) = 0
Dec(\kappa = 5, $ct$=2) = 1

There are six possible keys, only one of which gives $msg$=4. Thus P[$msg$ = 4 | $ct$ = 2] = 1/6. P[ $msg$ = 4 ] is the reciprocal of the size of the message space, which is also 1/6.
