The syntax of programming languages is similar to that of natural languages, with recursive structures allowing for statements of arbitrary length and complexity. An early method for describing the structure of programming languages is the ``Context-Free Grammar," developed by Noam Chomsky for natural languages. However, this framework, while robust enough to describe most programming languages in use today, has some unfortunate asymptotic downsides.

The classical method for CFG parsing is the Cocke-Younger-Kasami algorithm, which takes O(|G|n^3) time, where n is the length of the statement and |G| is the number of rules in the CFG. "Fast Context-Free Grammar Parsing," by Lillian Lee (P' #1), establishes a profound link between CFGs and matrix multiplication. She proves that any algorithm that decides whether a string is valid under some CFG with time complexity O(|G|n^{3 - \epsilon}) can be converted to a boolean matrix multiplication algorithm that runs in O(m^{3 - \epsilon}), where n and m differ by a constant factor. This sets a lower bound on CFG parser complexity to that of matrix multiplication, for which even the fastest algorithms are more than quadratic.

This limitation does not mean that linear-time parsing of programming languages is impossible, it merely means that they must be organized in a less complex way. An early solution is LL(k) grammars, which always use at maximum k tokens of look-ahead to parse a statement. These, however, can lead to ambiguity, with multiple possible correct parse-trees for the same statement.
